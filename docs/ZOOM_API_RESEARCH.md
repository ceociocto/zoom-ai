# Zoom API/SDK 实时转录方案调研报告

## 概述

本文档调研了 Zoom 官方 API/SDK 提供的实时转录能力，并与当前项目采用的开源方案进行对比分析，为技术选型提供参考。

**调研日期:** 2026-03-31
**调研目的:** 评估 Zoom 官方方案是否可以作为当前免费实时转录方案的替代或补充

---

## 一、Zoom 官方方案调研

### 1.1 Zoom Video SDK - Live Transcription

Zoom Video SDK 是面向开发者的实时音视频通信解决方案，提供内置的实时转录功能。

#### 架构设计

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         Zoom Video SDK 架构                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────┐      ┌─────────────────┐      ┌─────────────┐      │
│   │  自定义应用  │─────▶│ Zoom Video SDK  │─────▶│  Zoom 云端  │      │
│   │  (Web/移动) │      │   (嵌入式)      │      │  (AI 转录) │      │
│   └─────────────┘      └─────────────────┘      └─────────────┘      │
│                                 │                         │            │
│                                 │         ┌───────────────┘            │
│                                 ▼         ▼                          │
│                          ┌─────────────────┐                          │
│                          │  实时转录回调    │                          │
│                          │  (JSON 数据流)  │                          │
│                          └─────────────────┘                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 核心功能

| 功能 | 描述 | 支持情况 |
|------|------|----------|
| **实时转录** | 会议中实时生成字幕 | ✅ |
| **多语言支持** | 支持多种语言的转录 | ✅ 包括中文 |
| **自定义词汇** | 可添加专业术语提升准确率 | ✅ |
| **说话人识别** | 区分不同发言人 | ❌ 需额外付费 |
| **隐私模式** | 不保存音频数据 | ✅ |

#### 代码示例 (JavaScript)

```javascript
import { ZoomVideoSdk } from '@zoom/videosdk';

const client = ZoomVideoSdk.createClient();

// 订阅实时转录事件
client.on('caption-message', (data) => {
  const { speakerId, captionText, timestamp } = data;
  console.log(`[${speakerId}]: ${captionText}`);
});

// 开启实时转录
await client.liveTranscription.start();
```

#### 定价模式 (2026)

| 套餐 | 价格 | 实时转录 | 限制 |
|------|------|----------|------|
| **免费版** | $0/月 | ❌ 不支持 | 10,000 分钟/月 |
| **按量付费** | $0.0035/分钟/用户 | ✅ 支持 | 最低 $100/月 |
| **企业定制** | 联系销售 | ✅ 支持 | 包含说话人识别 |

**结论:** 免费版不支持实时转录 ❌

### 1.2 Zoom Meeting SDK - Closed Captioning

Zoom Meeting SDK 允许将 Zoom 会议功能嵌入第三方应用。

#### 实时字幕机制

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      Zoom Meeting SDK 字幕流程                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────┐      ┌─────────────────┐      ┌─────────────┐      │
│   │  Zoom 会议  │─────▶│  自动转录服务   │─────▶│  字幕流     │      │
│   │  (云端)     │      │  (付费功能)     │      │  (实时)     │      │
│   └─────────────┘      └─────────────────┘      └─────────────┘      │
│                                                         │              │
│                                                         ▼              │
│                                                   ┌─────────────┐     │
│                                                   │  SDK 回调    │     │
│                                                   │  事件监听    │     │
│                                                   └─────────────┘     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 使用前提

1. **许可证要求:** 需要 Pro、Business 或 Enterprise 许可证
2. **主持人权限:** 需要会议主持人开启自动转录功能
3. **账户限制:** 免费账户（Basic）无法使用

#### 代码示例

```javascript
// 监听字幕事件
zoomMeeting.on('caption', (data) => {
  console.log('字幕:', data.captionText);
  console.log('说话人:', data.speakerName);
});

// 获取历史字幕
const captions = await zoomMeeting.getCaptions();
```

**结论:** 免费账户无法使用 ❌

### 1.3 Zoom REST API - 录制转录

通过 Zoom REST API 获取会议录制文件，然后进行转录。

#### 工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Zoom 会议  │────▶│  云端录制    │────▶│  下载文件    │────▶│  本地转录    │
│  (结束)     │     │  (MP4/M4A)  │     │  (高延迟)    │     │  (Whisper)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  智能录制    │
                    │  (Transcript)│
                    │  API 可获取   │
                    └─────────────┘
```

#### 关键发现

| 能力 | Zoom REST API | 说明 |
|------|---------------|------|
| **实时音频流** | ❌ 不支持 | 无实时音频接口 |
| **录制文件下载** | ✅ 支持 | 会议结束后可用 |
| **智能转录文本** | ✅ 支持 | 需要付费许可 |
| **实时字幕** | ❌ 不支持 | 只能通过 WebSocket |

**结论:** 无法实现实时转录 ❌

## 二、当前开源方案分析

### 2.1 方案概览

本项目采用多种开源技术组合，实现完全免费的实时转录功能。

#### 架构设计

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        当前开源方案架构                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────┐      │
│   │                      Zoom Web 客户端                        │      │
│   └─────────────────────────────────────────────────────────────┘      │
│                          │                    │                       │
│              ┌───────────┘                    └───────────┐           │
│              ▼                                              ▼           │
│   ┌─────────────────────┐                    ┌─────────────────────┐   │
│   │   方式 A: DOM 监听   │                    │   方式 B: 音频捕获   │   │
│   │   Playwright        │                    │   系统音频           │   │
│   └─────────┬───────────┘                    └─────────┬───────────┘   │
│             │                                          │               │
│             ▼                                          ▼               │
│   ┌─────────────────────┐                    ┌─────────────────────┐   │
│   │   Zoom 原生字幕      │                    │   WhisperLiveKit    │   │
│   │   (需主持人开启)     │                    │   本地 ASR 引擎     │   │
│   └─────────┬───────────┘                    └─────────┬───────────┘   │
│             │                                          │               │
│             └─────────────────┬────────────────────────┘               │
│                               ▼                                        │
│                    ┌─────────────────────┐                            │
│                    │   字幕叠加渲染       │                            │
│                    │   (虚拟摄像头输出)    │                            │
│                    └─────────────────────┘                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 技术组成

| 组件 | 技术选型 | 功能 | 许可证 |
|------|----------|------|--------|
| **浏览器控制** | Playwright | 控制 Zoom Web、读取 DOM | Apache-2.0 |
| **音频捕获** | sounddevice | 系统音频实时捕获 | MIT |
| **流式转录** | WhisperLiveKit | 超低延迟 ASR + 说话人识别 | MIT |
| **字幕渲染** | PIL + OpenCV | 中文支持 + 样式渲染 | PSF/BSD |

### 2.3 性能指标

| 指标 | 当前方案 | 说明 |
|------|----------|------|
| **转录延迟** | < 500ms | WLK 流式处理 |
| **说话人识别** | ✅ 支持 | 基于 NeMo |
| **中文支持** | ✅ 原生 | Whisper 模型 |
| **并发实例** | 多实例 | 单服务器运行多个 Bot |
| **月度成本** | $0 | 完全免费 |

## 四、技术选型建议

### 4.1 推荐使用当前开源方案的场景

✅ **适合以下情况：**

- 预算有限的个人开发者或初创团队
- 需要说话人识别功能
- 对数据隐私有要求（本地处理）
- 需要深度自定义功能
- 技术团队有能力维护开源方案
- 能够接受稍高的技术复杂度

### 4.2 推荐使用 Zoom Video SDK 的场景

✅ **适合以下情况：**

- 有付费预算的商业项目
- 需要官方 SLA 保障
- 团队技术能力有限，希望快速接入
- 不需要说话人识别等高级功能
- 对 Zoom 生态有深度依赖
- 需要自定义词汇表功能

## 五、结论

### 5.1 核心发现

1. **Zoom 免费方案不支持实时转录**
   - Video SDK 免费版不包含转录功能
   - Meeting SDK 需要付费许可证
   - REST API 无实时音频流接口

2. **当前开源方案功能更全面**
   - 完全免费
   - 支持说话人识别（Zoom 需额外付费）
   - 本地处理，数据隐私更好
   - 完全可定制

3. **Trade-off 明确**
   - 开源方案：免费但需自行维护
   - Zoom SDK：付费但官方支持

### 5.2 最终建议

**对于本项目，建议继续使用当前开源方案：**

| 原因 | 说明 |
|------|------|
| **成本优势** | 每年节省 $1,300+ |
| **功能优势** | 免费获得说话人识别 |
| **灵活性** | 完全可控，可自由扩展 |
| **已验证** | 当前方案已稳定运行 |

**未来考虑迁移到 Zoom SDK 的条件：**

- 项目达到商业规模
- 愿意承担运营成本
- 需要官方技术支持
- 需要与 Zoom 深度集成
